The CongoCC project began in early 2022 as a rebranding of the JavaCC 21 project, which itself was a ground-up rewrite of the earlier JavaCC project, originally developed at Sun Microsystems in the 1990's.

Here are a few highlights in the history of JavaCC, FreeCC, and CongoCC:

  • 1996: JavaCC developed and released internally at Sun Microsystems and at a startup company Metamata. Based on Java v1.
  • 1997: JJTree preprocessor added to JavaCC codebase, implemented by a different person (Rob Duncan) than those who developed the original JavaCC
  • 1998: JavaCC acquired by Webgain but was still available for free download
  • 2001: Webgain falls victim to 2001 dot-com crash and JavaCC becomes an asset of Sun Microsystems again
  • 2003: To much fanfare, Sun Microsystems releases JavaCC (and JJTree) as open source.
  • 2008: JavaCC was forked as the FreeCC project. FreeCC leveraged the FreeMarker template engine (that also was based on JavaCC) to simplify and amplify the code generation capabilities of FreeCC.
    • Introduced INCLUDE statement that allows a large grammar to be broken up into multiple physical files
    • Introduced INJECT statement in grammar files allow Java code to be added to any generated file. Inject statements can be added to grammar file close to where their relevant productions
    • FreeCC unified the JavaCC and JJTree preprocessor into a single tool, thus simplifying drastically the workflow of a typical JavaCC-based project
    • Building an AST Tree is the default, though it can be turned off if desired
  • 2009 - 2019: Development hiatus
  • 2020 - 2022 : Development resumes, now rebranded as JavaCC 21. As of May 2021, the codebase was completely rewritten. Changes and improvements include:
    • Streamlined grammar syntax that removed a lot of the visual clutter in the legacy syntax. (The older syntax is still supported.)
    • Preprocessor that implements #define/#undef/#if/#elif/#else/#endif in the same way as the C# preprocessor
    • Contextual predicates provide a way of checking whether we are at a given point in a parse
    • Improved lookahead capabilies available in new SCAN keyword, including the up-to-here marker =>||
    • Nested syntactic lookahead finally works!
    • Fault-tolerant parsing that incorporates heuristics for building an AST with "dirty" or "incomplete" Nodes
    • ATTEMPT/RECOVER statement to replace try/catch blocks to produce more meaningful fallback or recovery actions when the attempt fails.
    • Python 3.10 grammar/parser available for your own projects
    • C# grammar/parser available for your own projects
    • Support for generating parsers in Python and C# as well Java. (The work of Vinay Sajip)
  • 2022: JavaCC 21 rebranded as CongoCC
